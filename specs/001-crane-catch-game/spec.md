# Feature Specification: クレーンキャッチゲーム（Crane Catch Game）

**Feature Branch**: `001-crane-catch-game`  
**Created**: 2026-03-17  
**Status**: Draft  
**Input**: User description: "ゲームセンターにあるクレーンキャッチャーゲームを再現した、5〜10歳の子供向け教育ゲーム。クレーンを操作してさまざまなアイテムをキャッチし、コレクションとして図鑑に登録していく。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - クレーンでアイテムをキャッチする (Priority: P1)

子供がクレーンを上下左右に動かして2D平面上で位置を決め、ボタンを押してリングを下ろし、ゲーム内のアイテム（どうぶつ）をキャッチする。キャッチに成功するとお祝い演出が表示され、アイテムが図鑑に登録される。失敗した場合も励ましのメッセージが表示され、再挑戦できる。

**Why this priority**: ゲームの核心体験。クレーン操作とキャッチの楽しさがなければゲームとして成立しない。

**Independent Test**: クレーン画面を開き、タッチ操作でクレーンを上下左右に移動→ボタンタップでリング降下→アイテムキャッチ成功/失敗の一連のフローが完結する。

**Acceptance Scenarios**:

1. **Given** クレーンゲーム画面が表示されている, **When** 子供がクレーンを任意の方向（上下左右）にスワイプする, **Then** クレーンがスワイプした方向にスムーズに移動する
2. **Given** クレーンが2D平面上の任意の位置にある, **When** 子供がキャッチボタンをタップする, **Then** リングが下降しアイテムの掴み判定が行われる
3. **Given** リングがアイテムの上にある, **When** リングが下降してアイテムを包み込むように掴む, **Then** アイテムが持ち上げられ、お祝い演出（音・エフェクト）が再生される
4. **Given** リングがアイテムから外れた位置にある, **When** リングが下降する, **Then** 何も掴めず、励ましメッセージ（「おしい！もういっかい やってみよう！」等）が表示される
5. **Given** キャッチに成功した, **When** 演出が完了する, **Then** 結果画面に遷移し、キャッチしたアイテムの名前と3Dモデルが表示される

---

### User Story 2 - 図鑑でコレクションを閲覧する (Priority: P2)

子供がこれまでにキャッチしたアイテムを図鑑画面で閲覧する。キャッチ済みのアイテムは3Dモデルが回転表示され、ひらがなの説明文を読むことができる。未取得アイテムはシルエットで表示され、コンプリートへの意欲を促す。

**Why this priority**: コレクション要素がリプレイ動機となり、教育的価値（どうぶつの知識）を提供する。

**Independent Test**: 図鑑画面を開き、キャッチ済みアイテムの3Dモデル表示・説明文閲覧・未取得アイテムのシルエット表示を確認できる。

**Acceptance Scenarios**:

1. **Given** 図鑑画面を開いた, **When** キャッチ済みアイテムをタップする, **Then** アイテムの3Dモデルが回転表示され、ひらがなの説明文が表示される
2. **Given** 図鑑画面を開いた, **When** 未取得アイテムを確認する, **Then** シルエット（？マーク付き）で表示され、タップしても詳細は見られない
3. **Given** カテゴリ内の全アイテムをキャッチ済み, **When** 図鑑画面を開く, **Then** コンプリート演出が表示される

---

### User Story 3 - カテゴリを選択してゲームを開始する (Priority: P3)

子供がタイトル画面から遊びたいカテゴリ（最初は「どうぶつ」）を選び、クレーンゲームを開始する。カテゴリ選択画面では各カテゴリのアイコンと名前がひらがなで表示される。

**Why this priority**: ゲームの導線として必要。将来のカテゴリ拡張の基盤となる。

**Independent Test**: タイトル画面→カテゴリ選択画面→クレーンゲーム画面の遷移が完結する。

**Acceptance Scenarios**:

1. **Given** タイトル画面が表示されている, **When** 「あそぶ」ボタンをタップする, **Then** カテゴリ選択画面に遷移する
2. **Given** カテゴリ選択画面が表示されている, **When** 「どうぶつ」カテゴリをタップする, **Then** どうぶつカテゴリのクレーンゲーム画面に遷移する
3. **Given** カテゴリ選択画面が表示されている, **When** 未解放カテゴリを確認する, **Then** 「きんきゅう（近日公開）」等の表示がされ、タップしても開始できない

---

### User Story 4 - はじめてのチュートリアル (Priority: P4)

初回プレイ時に操作方法を案内するチュートリアルが表示される。クレーンの動かし方、ボタンの押し方をステップバイステップで体験しながら学べる。

**Why this priority**: 対象年齢が低いため、直感的に操作を覚えられるガイドが必要。

**Independent Test**: 初回起動時にチュートリアルが自動表示され、指示に従って操作を完了できる。

**Acceptance Scenarios**:

1. **Given** ゲームを初めて起動した, **When** クレーンゲーム画面に入る, **Then** チュートリアルが自動的に開始される
2. **Given** チュートリアル中, **When** 「くれーんを うごかしてみよう！」と指示が出る, **Then** 子供がクレーンを上下左右いずれかに動かすと次のステップに進む
3. **Given** チュートリアルが完了した, **When** 次回以降ゲームを起動する, **Then** チュートリアルはスキップされる（設定から再表示可能）

---

### User Story 5 - タイトル画面の表示と導線 (Priority: P5)

ゲーム起動時にタイトル画面が表示される。ゲームタイトル、「あそぶ」ボタン、「ずかん」ボタンが配置され、ポップで楽しい雰囲気が演出される。

**Why this priority**: ゲームの入口。シンプルだが全体の導線の起点。

**Independent Test**: 起動時にタイトル画面が表示され、各ボタンから対応する画面に遷移できる。

**Acceptance Scenarios**:

1. **Given** ゲームを起動した, **When** ロードが完了する, **Then** タイトル画面にゲームタイトル・「あそぶ」ボタン・「ずかん」ボタンが表示される
2. **Given** タイトル画面が表示されている, **When** 「ずかん」ボタンをタップする, **Then** 図鑑画面に遷移する

---

### Edge Cases

- クレーン操作中に画面外にスワイプした場合 → クレーンは2D可動範囲（上下左右）の端で停止する
- キャッチボタンを連打した場合 → アーム動作中は追加入力を無視する
- リングがアイテムを掴んだが途中で落とした場合 → キャッチ失敗として扱い、励ましメッセージを表示する
- ブラウザのタブが非アクティブになった場合 → ゲームを自動的に一時停止する
- ローカルストレージが利用できない/満杯の場合 → セッション内のプレイは継続可能、保存失敗時はユーザーに通知する
- 端末を縦向きにした場合 → 横向きへの回転を促すメッセージを表示する
- 同じアイテムを再度キャッチした場合 → 図鑑には重複登録せず、「もう つかまえたよ！」等のメッセージを表示する

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: ゲームはタイトル画面→カテゴリ選択→クレーンゲーム→結果画面→図鑑の画面遷移をサポートしなければならない
- **FR-002**: クレーンはタッチ操作（上下左右スワイプまたはボタン）で2D平面上（X-Y平面）を移動できなければならない
- **FR-003**: キャッチボタンをタップするとリング（丸い輪っか形状）が下降し、アイテムを包み込むように掴む当たり判定を行わなければならない
- **FR-004**: キャッチ成功時にお祝い演出（視覚エフェクト＋サウンド）を再生しなければならない
- **FR-005**: キャッチ失敗時に励ましメッセージを表示しなければならない（ネガティブな表現は使用しない）
- **FR-006**: キャッチ成功したアイテムはローカルに永続保存し、次回起動時にも図鑑に反映されなければならない
- **FR-007**: 図鑑画面ではキャッチ済みアイテムの3Dモデルを回転表示し、ひらがなの説明文を表示しなければならない
- **FR-008**: 未取得アイテムは図鑑上でシルエット表示しなければならない
- **FR-009**: 初回プレイ時に操作チュートリアルを表示しなければならない
- **FR-010**: カテゴリ選択画面で利用可能なカテゴリを一覧表示しなければならない
- **FR-011**: 初期リリース時に「どうぶつ」カテゴリ（ライオン、ゾウ、キリン、ペンギン、パンダ、コアラ、イルカ、ウサギの8種）を提供しなければならない
- **FR-012**: アイテムの3Dモデルはプロシージャルに生成しなければならない（外部3Dファイル不要）
- **FR-013**: BGMと効果音はプログラムで生成しなければならない（外部音声ファイル不要）
- **FR-014**: すべてのUIテキストはひらがな中心で表示しなければならない（対象年齢5〜10歳）
- **FR-015**: 横向き（ランドスケープ）表示に最適化しなければならない
- **FR-016**: 端末が縦向きの場合、横向きへの回転を促すメッセージを表示しなければならない
- **FR-017**: タッチ操作に適した大きなボタンサイズ（最小44×44ポイント以上）を採用しなければならない
- **FR-018**: 新カテゴリの追加時にコアゲームロジックの変更が不要なデータ駆動設計でなければならない
- **FR-019**: クレーンゲーム内のアイテム配置は2Dグリッド（X-Y平面）上でプレイごとにランダム化しなければならない
- **FR-020**: カテゴリ内の全アイテムをコンプリートした際に特別な演出を表示しなければならない

### Key Entities

- **カテゴリ（Category）**: ゲーム内のアイテムグループ。名前（ひらがな）、アイコン、解放状態、所属アイテムリストを持つ。将来的に追加可能。
- **アイテム（Item）**: キャッチ対象のオブジェクト。名前（ひらがな）、カテゴリ所属、3Dモデル定義パラメータ、図鑑説明文、レアリティを持つ。
- **図鑑（Collection/Zukan）**: プレイヤーのコレクション状態。アイテムごとの取得/未取得状態、取得日時を保持する。ローカルに永続化される。
- **クレーン（Crane）**: ゲーム内の操作オブジェクト。2D平面上の位置（X-Y座標）、リング形状のキャッチ機構、リング状態（待機/下降中/掴み中/上昇中）を持つ。リングは丸い輪っか形状で、開閉してアイテムを包み込むように掴む。
- **プレイセッション（Play Session）**: 1回のゲームプレイ。選択カテゴリ、出現アイテムリスト、キャッチ結果を持つ。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 5歳の子供がチュートリアルなしでも3回以内の試行でクレーン操作の基本（移動+キャッチ）を理解できる
- **SC-002**: 1回のゲームプレイ（カテゴリ選択からキャッチ完了まで）が2分以内で完了できる
- **SC-003**: iPad Safari横向きで60FPSの滑らかな3D描画を維持できる
- **SC-004**: 初回ロードが3秒以内に完了する（外部アセットなしのため高速）
- **SC-005**: 「どうぶつ」カテゴリの8種類すべてのアイテムが図鑑に登録可能である
- **SC-006**: 図鑑の3Dモデル表示と説明文閲覧が直感的に行え、子供が自力で操作できる
- **SC-007**: 新カテゴリの追加がデータ定義の追加のみで可能であり、ゲームロジックの修正が不要である
- **SC-008**: キャッチ成功率が平均40〜60%の範囲に調整され、簡単すぎず難しすぎない体験を提供する
- **SC-009**: すべてのUIテキストが5歳児でも読めるひらがな中心の表記になっている
- **SC-010**: ネガティブな表現が一切なく、失敗時も前向きな励ましメッセージが表示される

## Assumptions

- プレイヤーはiPad Safari横向きでの利用が主な想定環境である
- インターネット接続は初回ロード時のみ必要で、ゲームプレイ中はオフラインでも動作する
- ゲームの進行データ（図鑑の取得状態）はブラウザのローカルストレージに保存する
- 課金要素・広告は一切含まない（純粋な教育・娯楽目的）
- BGMと効果音はWeb Audio APIでプロシージャル生成し、外部ファイルを使用しない
- 3DモデルもThree.jsのジオメトリでプロシージャル生成し、外部3Dファイルを使用しない
- デザインテイストはuniverse-kids-raceを踏襲するが、カラーパレットはよりポップな配色とする
- 初期リリースでは「どうぶつ」カテゴリのみ提供し、追加カテゴリは後続リリースで対応する
- 保護者の管理機能（プレイ時間制限等）は初期リリースのスコープ外とする
